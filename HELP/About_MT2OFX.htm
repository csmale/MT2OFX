Welcome to MT2OFX V3.5

This program allows the use of Microsoft Money and other personal finance software with downloaded statement files from a wide variety of banks.

 

Text-based formats (including Comma-separated values, CSV) can be supported by creating a script to parse and decipher the input file. Many formats are supported in the standard installation, including generic support for QIF and OFC as input formats.

 

The SWIFT MT940 format of statement download files is used by many banks, including many in Germany, Switzerland and the Netherlands.

 

The program works by translating the input files (in a supported text format including SWIFT MT940) into OFX, OFC or QIF format. Although the conversion is largely automatic, there are some ways of influencing the operation in order to achieve exactly the results you require. Through the Options Window the program can be configured as you wish. These parameters are held in the file MT2OFX.INI, in the same directory as the program (probably C:\Program Files\MT2OFX). This file can also also be viewed and modified using any text editor program such as Notepad.

 

MT2OFX can be started with its normal window, which allows you to select the input and output files and select options before initiating the conversion. In addition, by associating the .STA file type (or any other file type) with MT2OFX, the conversion can be carried out without any intervention, using options which you have pre-selected for this mode of operation. This file association is set up during installation for .STA and .940. Other file types must be associated by hand through Windows Explorer.

 

Program Flow

Once the conversion operation has been initiated, operation proceeds through the following stages:

1. If the input is coming from the Windows Clipboard, it is assumed to be a text file with an unknown extension; go to step 4.

2. If the extension of the file has been mapped directly to a script, load that script and go to step 11.

3. If the extension is one of the MT940 Extensions, goto step 6.

4. Pass the file to each of the text processing scripts in turn. If a script recognises the input file as its own, go to step 11.

5. If no script recognises the input file, it cannot be processed.

6. (For MT940 files) Identify the bank which issued the file

7. Load treatment options appropriate for this bank

8. Read the MT940 file, identifying Statements and Transactions

9. Iterate through the Statements and Transactions, deriving additional information required for the output on the way, possibly using additional logic in external scripts.

10. Go to step 12.

11. (For non-MT940 files) Use the script to read and process the input file.

12. Perform Payee Mapping (from version 3.1)

13. Write the output file in OFX, OFC or QIF format.

 

If you would like to set up MT2OFX for a bank which is not provided in the standard distribution of the product, see Supporting a New Bank.
