QIF Output

Starting with version 2.1 this program is also capable of producing output in QIF format.

 

Format Limitations

QIF is not such a rich format as OFX. Many data items in the MT940 input have no place in a QIF file.

 

Multiple Accounts in One File

A QIF file is normally expected to contain transactions from one single account. Quicken will handle multi-account files correctly, but Microsoft Money will not; all account headers after the first produce a warning, and all transactions are imported to the same account, regardless of the account headers in the file.

 

Quicken's Support for QIF

Intuit, the producers of Quicken and originators of the QIF format, have announced that they will no longer support QIF as an import/export format as from Quicken 2005. You are OK for a while!

 

How to make MT2OFX produce QIF files

Simply set QIF as the output file type in the Options dialog. For details, see the documentation for OutputFileType .

 

QIF-Specific Fields

The Category and Cleared Status fields can be set from the processing scripts. The standard scripts do not set these fields, but you may add your own logic to these scripts if you wish. Additionally, the Payee Mapping facility can be used to derive the Category automatically from the transaction data.

 

Transaction Splits

As of version 3.2 these are supported by the program's underlying data model. There is no automatic way of creating splits, but any which are present in a QIF input file are correctly propagated to the output. Processing scripts may create and manipulate splits if they are programmed to do so.

 

Custom QIF Fields

If you require any additional fields in the QIF output, these can be added by scripting. (From V3.4)
