Txn Object

The Txn object provides the following properties:

 

Name

Type

Description

Statement

Statement

(read-only) The Statement object which contains this transaction.

Index

Integer

(read-only) The index of this transaction within the statement. The first transaction has Index=1.

ClearedStatus

String

Value for the "C" line in QIF output.

Amt

Double

The amount of this transaction. A credit transaction is positive, a debit transaction is negative. Note: the currency is always the account currency, which can be found through any of the balances through the Statement object.

ValueDate

Date

The date on which the transaction is deemed to have occurred for interest calculations.

BookDate

Date

The date on which the transaction is posted to the account.

IsReversal

Boolean

True if this transaction is a reversal (correction) of an earlier transaction.

BookingCode

String

An indication of the type of the transaction. Although the MT940 defines standard values for this field, most banks use their own system of codes.

Reference

String

Customer reference

BankReference

String

Bank reference

Memo

String

The transaction description provided. (from version 3.1)

FurtherInfo

String

Another name for Memo, for compatibility with older scripts.

Category

String

The transaction category. Only used in QIF output! (from version 3.0.18)

SIC

Integer

Standard Industry Classification code. May be used by Money etc. to associate a Category with a transaction. OFX only. (from V3.4)

Splits

Splits

(read-only) Returns a collection of Split objects containing the category splits for this transaction.

Str86

Str86Info

(read-only) If the transaction contains a structured description in the :86: record.

NonSwift

NonSwiftInfo

(read-only) The structured information contained in the :NS: record associated with this transaction.

Payee

Payee

The payee for this transaction. (From V3.4; previously a String)

SkipPayeeMapping

Boolean

If True, this transaction will not be processed by Payee Mapping. (from version 3.1)

TxnDateValid

Boolean

True if TxnDate contains a valid date.

TxnDate

Date

The date on which the transaction physically took place. For example, point-of-sale transactions and cash withdrawals may be booked on a different date to when you were actually in the shop or ATM.

TxnType

String

Transaction type string which will be used in the OFX output. Must be valid for the <TXNTYPE> tag in OFX. Valid values are listed below.

FITID

String

Unique identification of this transaction.

CheckNum

String

Cheque number.
