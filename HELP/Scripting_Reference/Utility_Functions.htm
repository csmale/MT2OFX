Utility Functions

The following utility functions and procedures are available from the scripts. See also Text Handling Functions.

 

Function MakeGUID() As String

Returns a GUID (Globally Unique Identifier) - a string which is guaranteed to be unique. This can be useful for generating StatementIDs if the input file doesn't contain information to identify the statements uniquely.

 

Sub Message(ToScreen As Boolean, ToFile As Boolean, Text As String, Title As String)

Writes a message to the log file and/or to the screen.

 

Function GetConfigString(sSection As String, sKey As String, sDefault As String) As String

Returns a value from the configuration file MT2OFX.INI.

 

Function GetConfigStringEx(sSection As String, sKey As String, sDefault As String, sFile as String) As String

Returns a value from the configuration file given by sFile.

 

Function Version() As String

Returns a string representing the version number of the MT2OFX application, e.g. 2.2.127.

 

Function DecimalSeparator() As String

Returns a string containing the decimal separator character from the user's regional settings in Windows Control Panel.

 

Function UserCurrency() as String

Returns a string containing the ISO currency code associated with the user's chosen locale in Windows Control Panel.

 

Sub Abort()

Causes script processing to be terminated as soon as possible. For use in scripts which detect an unrecoverable error.

 

Function CheckVersion()

Checks the version of Windows Scripting installed on your computer. If it is V5.6 or newer (required for MT2OFX) then the function returns True. Otherwise an explanatory message is given and the function returns False.

 

Function GetScriptEngineInfo()

Returns a string containing the language, version and build number of the installed scripting engine. For example: "VBScript Version 5.6.300".

 

Sub LoadProperties(ScriptName, aPropertyList)

The property set named "ScriptName" is loaded for this script from persistent storage. "aPropertyList" describes the properties.

 

Function ShowConfigDialog(ScriptName, aPropertyList) As Boolean

The standard dialog for manipulating script property values is displayed using the currently loaded property set described by "aPropertyList". The "ScriptName" is used only for display purposes in the dialog. Don't forget to load the property set first!

 

Sub SaveProperties(ScriptName, aPropertyList)

The currently loaded script property set described by "aPropertyList", is saved to persistent storage under  "ScriptName".

 

Function GetProperty(PropertyName) As Variant

The value of the property "PropertyName" is returned from the currently loaded script property set.

 

Function ChooseFromList(sList As Variant, sDefault As Variant, sTitle As Variant, sHelp As Variant, bFixedList As Variant) As String

Displays a dialog to allow the user to select one value from a list. The values must be provided in sList, separated by "\". (From V3.4)

 

NODATE

A constant Date value which represents no valid date, to indicate that the date is unknown.
