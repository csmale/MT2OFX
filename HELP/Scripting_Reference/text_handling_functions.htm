Text Handling Functions

These functions are available from within scripts to support the processing of text input files. See also Utility Functions.

 

Function ReadLine() As String

Returns the next line from the input file.

 

Function AtEOF() As Boolean

Returns True if the end of the input file has been reached, i.e. a further call to ReadLine() will cause an error.

 

Sub Rewind()

Resets the file pointer of the input file to the start of the file.

 

Function EntireFile() As String

Returns the entire input file as a long string. (From V3.4)

 

Function NewTransaction() As Txn

Returns a new Txn object in which a transaction can be stored. The new transaction is automatically added to the Statement.Txns collection.

 

Function NewStatement() As Statement

Returns a new Statement object. The new Statement is automatically added to the Session.Statements collection.

 

Function ParseLineFixed(ByVal Line As Variant, ByVal Pattern As Variant) As Variant

Returns an array of strings (1-based) representing the parts of the input line captured when it is matched against the pattern using the RegExp functionality in VBScript version 5 and later.

 

For full details of the syntax of a pattern, please consult the documentation included in Microsoft Scripting 5.6 which can be consulted online. The documentation can be downloaded here.

 

Function ParseLineDelimited(ByVal Line As String, ByVal Delimiter As String) As Variant

Returns an array of strings (1-based) representing the parts of the input line delimited by the given delimiter, typically a comma or a tab character. The fields may also be enclosed in single or double quotes which are removed before the value is returned.

 

Function ParseNumber(NumString As String, DecimalChar As String) As Double

Parses the string in NumString, which must represent a number using DecimalChar as decimal point. The number string must contain only digits, "-", "+", "." and ",".
