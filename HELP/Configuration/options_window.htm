MT2OFX V3.5 Help

 

2011-06-21

MT2OFX Options Window

The options window is where the basic program configuration is defined. Click on an item in the picture below to learn more about it.

 

 

If you have modified any of the options on this screen, click on OK to save the state of the program options (see below).

 

The program has many options which affect its behaviour. All options are configurable from MT2OFX.INI; some useful options can be selected in this window before the program actually does its work. When MT2OFX starts, this window displays the options selected in MT2OFX.INI. If you change the options in this window, the changes can be saved back to MT2OFX.INI by clicking on OK.

 

The program options which are selectable here are as follows.

 

If Show Open Dialog on startup is checked, you will be prompted to select an input file as soon as the program starts, as if you had clicked on the "Browse" button in the main window.

 

If Start import after conversion is checked, then the output file will be passed to the program which has been configured in Windows to handle that type of files. For example, if you use Microsoft Money, OFX files are associated with mnyimprt.exe. If this option is not checked, the import will not be started automatically, but a dialog will be displayed which allows you to choose whether or not to import the file at this time.

 

The Save output: setting determines where the result of the conversion is stored. The choices are as follows:

Ask - for each file processed - before the conversion takes place you will be presented with a Save As dialog, allowing you to choose the name and location of the output file.

Automatic - same directory as input - the output file will be saved under the same name as the input, with the extension set to the configured Default Output Type.

Temporary - deleted after use - the output will be created in a temporary file, with the extension set to the configured Default Output Type. After the output file has been "imported" into your financial software, the temporary file will be deleted.

 

Checking the Save output from Clipboard conversion option will save allow you to save the result of a conversion of data from the clipboard. You will be presented with a Save As dialog, allowing you to choose the name and location of the output file. When this option is unchecked, the output from clipboard conversion is saved in a temporary directory and deleted after import.

 

If Compress multiple spaces in payee and memo is checked, multiple consecutive spaces in the Payee and Memo fields will be replaced by a single space. This will tend to make these values more easily readable.

 

As from version 3.1, MT2OFX will not normally generate a cheque number automatically if the processing script fails to provide one. Check Always generate cheque number to revert to the old behaviour, where a missing cheque number is filled in automatically by combining the last few characters of the statement identifier with a sequence number.

 

Normally the transactions in the output file will be sorted into ascending date order. By checking Do not sort transactions into date order the transactions will be in the same order in the output file as they occur in the input file.

 

By checking Skip Empty Statements MT2OFX will ignore statements which contain no transactions.

 

Microsoft Money looks only at the so-called Book Date of a transaction, which is the date on which the transaction is processed by the bank. Other dates are associated with a transaction, including the Value Date (which is used in interest calculations) and the actual date of the transaction. If you prefer to see one of the other dates in Microsoft Money, you can select it by using the Use <date> as book date selection.

 

OutputFileType defines the default file extension of the output file, and hence the data format (OFC, OFX or QIF).  If you use Quicken, use QIF; for other programs, use OFX, OFC or QIF. All file extensions are OFX formatted, except for OFC and QIF. It is this extension which is used when MT2OFX is run invisibly having been associated in Windows with an input file type.

 

OFX Version allows selection of the version of the OFX specification which is used in OFX output files. Normally there is no need to change the default (102), but if you have a need, the version can be set to 102, 151, 160 or 201, which are the published versions of the OFX Specification.

 

MemoDelimiter defines the character which is used to separate data from different fields or different lines in the transaction description. OFX and most financial software allow only a single line of text, whereas the original MT940 format supports multiple lines of text.

 

The PayeeCase setting affects the presentation of the payee name. Most banks only use upper case in downloaded statements.

 

QIF DateFormat represents the format of dates in the QIF output. Some programs require US-format dates, others require European dates or are more flexible. A custom format can also be used.

 

If the QIF date format is set to "Custom", the format to be used can be entered in QIF Custom Date Format.

 

QIF Decimal Separator defines the character which is used as the decimal separator in amounts output in a QIF file.

 

If you check the No !Account Header in QIF option, MT2OFX will not add a header section in QIF output files which describes the account for the subsequent transactions. Microsoft Money cannot handle these headers and may require this option to be set. Other software may be able to use this header to route the transactions directly to the appropriate account.

 

The option QIF Maximum Memo Length configures the number of characters allowed in the Memo field of QIF output. The field value will be truncated at this length. The default value of zero indicates that the Memo field will not be truncated.

 

The Script Editor is the program which is launched to edit a script file when you click on the Edit button. Click on the folder icon to choose the program you would like to use for editing .vbs files.

 

MT940 File Extensions contains a list of file extensions which are automatically assumed to contain MT940 data. Separate multiple values with commas, and do not include the "dot" before the extension. This information is also used when "MT940 Statement Files" in the "Browse" dialog.

 

Text File Extensions contains a list of file extensions which are automatically assumed to NOT contain MT940 data. Separate multiple values with commas, and do not include the "dot" before the extension. This information is also used when "Text Statement Files" in the "Browse" dialog.

 

Default Payee Name is the string which is used as the payee name in the output file when the processing script does not provide a payee for a transaction.
