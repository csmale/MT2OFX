Payee Mapping Window

This is where you can configure MT2OFX to translate payee names and assign a Category to the transaction.

 

Note that the OFX and OFC file formats do not allow a Category to be specified, so this information will effectively be ignored if you are not using QIF output.

 

 

For each transaction, the Payee or the Memo field (as defined in the rule) is matched against each pattern in turn, stopping when a match occurs. When a match is detected, the Payee field is replaced (if the replacement is not empty) and the Category is set (if the category string is not empty).

 

A transaction which doesn't match any of the patterns, is left unchanged.

 

To learn how to construct the patterns which the payee is matched against, refer to the Regular Expression Syntax information.
