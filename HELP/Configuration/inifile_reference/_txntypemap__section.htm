[TxnTypeMap] Section

This section enables the program to convert the four-character transaction codes in the MT940 transactions to OFX transaction types. If the program is unable to find an explicit translation, "OTHER" will be used.

 

The following table lists the transaction types. The Amount column indicates whether the transaction type is valid with a credit (+) or a debit (-) transaction. Note that these are the only transaction types which are valid in OFX.

Each entry describes a translation from a transaction code from the MT940 file to an OFX transaction type. If two OFX transaction types are given separated by a comma, the first value is used if the transaction amount is greater than or equal to zero and the second value is used if the value is less than zero.
