[QIF] Section

In this section specific details of the programs behaviour during production of QIF files can be configured.

 

DateFormat

Represents the format of dates in the QIF output. Some programs require US-format dates, others require European dates or are more flexible. Allowed values are:

 

0=mm/dd/yyyy - US format

1=dd/mm/yyyy - European format

2=yyyy/mm/dd - International Standard format

3=as defined in the Regional Settings in Windows Control Panel (default).

 

DecimalSeparator

Defines the character which is used as the decimal separator in amounts output in a QIF file. Typical values are "." (123.45), "," (123,45) or "". If the value is empty or missing, the setting is taken from the Regional Settings in Windows Control Panel (default).

 

NoAccountHeader

Indicates whether the output QIF file is to contain a header section describing the account to which the subsequent transactions apply. If the value is "0" or omitted, then an account header is produced; otherwise no account header is produced.

 

The account header looks like the following example:

!Account

N<account number>

T<account type>

DOFX online account

/<closing balance date>

$<closing balance amount>

^

 

The lines referring to the closing balance are omitted if the closing balance is not available.

 

Microsoft Money, and possibly other software, objects to the account header. Hence a facility has been provided to disable the header when required.
