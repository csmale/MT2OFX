Bank-Specific Configuration

In spite of the fact that MT940 is a defined standard, banks use differing interpretations of the official standard which is controlled by SWIFT.

 

Each bank identified in the [BankRules] Section, needs to have its own section in which the specific treatment is defined.

 

[bank]

BankName=bank_name

Structured86=0

FindPayee=ABNA_FindPayee

FindTxnDate=ABNA_FindTxnDate

FindServerTime=ABNA_FindServerTime

IntuitBankID=00508

ScriptFile=ABN Amro.vbs

 

 

 

# bank variant configuration. The section name must be equal to the bank name, which is matched

# against the rules given above. If an exact section name

# match is not found, the default is used which is defined under "[*]".

# BankName: String (not currently used)

# Structured86: 1=the :86: record is structured as described in the

#                SWIFT standard, otherwise it is unstructured

# FindPayee: The name of the internal function used to extract the

#            payee name from the description in :86:

# FindTxnDate: The name of the internal function used to extract the

#            transaction date from the description in :86:

# FindServerTime: The name of the internal function used to derive the

#            server timestamp from the file

# IntuitBankID: The identifier given to the bank by Intuit if the bank

#      has signed up to support Quicken. There are rumours that

#            Quicken will not import an OFX file unless the <INTU.BID>

#            field is present and correct!

# ScriptFile: The name of the file containing VBScript code used to postprocess

#            the transaction. If present, this code is used INSTEAD of the internal

#            functions mentioned above. For details of the scripting capabilities

#            see the Help information in mt2ofx.chm.

# SkipEmptyMemoFields: 1=Empty fields in the :86: data are ignored, thus avoiding

#            consecutive, leading and/or trailing delimiters in the output.
