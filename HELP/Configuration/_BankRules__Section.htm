[BankRules] Section

In order to work out which bank issued the input file, we match the initial lines against the patterns in the [BankRules] section. The rules are checked in order; as soon as a rule matches, that bank is assumed. Checking stops when a rule "Bank#" is not found. If no bank has been identified, the conversion is aborted and a message is given.

 

The rules contain 3 fields, separated by commas:

1)  The bank code used to find the section in this file with the appropriate configuration. By convention the SWIFT BIC (Bank Identifier Code) is used.  

2)  How many lines at the start of the file are checked before the file is deemed not to match.  

3)  The pattern, which is used to match against the entire line. Use a * to match any string, or ? to match any single character.  

 

When a bank has been determined, the bank code is used to load the Bank-Specific Configuration.

 

Examples:

[BankRules]

Bank1=ABNANL2A,1,ABNANL2A

Bank2=SNSBNL2A,1,{1:F01BBSPNL2A*

Bank3=FVLBNL22,5,:25:FVLBNL00*

Bank4=RABONL2U,1,:940:

Bank5=FTSBNL2R,1,:20:99 9999999999
