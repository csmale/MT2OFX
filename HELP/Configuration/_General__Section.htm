[General] Section

 

In the [General] section of MT2OFX.INI, parameters can be configured which govern the overall operation of the program.

 

MemoDelimiter

Defines the character which is used to separate data from different fields or different lines in the transaction description. OFX and most financial software allow only a single line of text, whereas the original MT940 format supports multiple lines of text.

OutputFileType

Defines the file extension of the output file, and hence the data format (OFC or OFX).  If you use Quicken, use QFX; for other programs, use OFX or OFC. All file extensions are OFX formatted, except for OFC.

StartImportAfterConversion

If this value is set to 1, the process of importing the resulting OFX/OFC file into the financial software is started automatically as soon as the conversion has finished. Otherwise the user is asked to confirm whether the import is to be started.

ApplicationHomePage

Defines the URL of the MT2OFX home page, which is displayed in the About dialog.

PromptForOutput

If this value is set to 1, the user is allowed to select the name for the output file (by means of a "Save As" dialog). Otherwise, the output file name is equal to the input file name, with the extension changed to the value of OutputFileType.

CompressSpaces

If this value is set to 1, consecutive space characters in the Payee name and transaction description are replaced with a single space.

PayeeCase

This setting affects the presentation of the payee name. Most banks only use upper case in downloaded statements. This can appear a little overpowering.

0=No change (default), 1=Upper Case, 2=Lower Case, 3=Proper Case

AutoBrowse

If this value is set to 1, a "File Open" dialog will be presented when the program is started in GUI mode. Otherwise, click the "Browse" button in the main window to select an input file.

BookDateMode

Microsoft Money, in common with many programs, uses only the Book Date of the transaction, and ignores the Value Date and the Transaction Date. This option allows you to configure which date is used in the Book Date field in the output file and thus which date is displayed.

0=Book Date (Standard, default), 1=Value Date, 2=Transaction Date (if available, else Book Date)

LogFile

File to which a simple log is written. Default is no logging.

TxnDumpFile

File to which a dump is written of all statements and transaction data. Must be called from the processing script!

ScriptTimeout

Number of milliseconds to allow for a script function to complete. If the value -1 is specified, no timeout is used. When the value is 0, the timeout occurs as soon as it is determined that a running script is hung for any reason. Default is 5000 (equivalent to 5 seconds).
