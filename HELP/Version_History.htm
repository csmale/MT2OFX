V2.0.15, 1 July 2003: First release of Version 2.0.

Important Changes in Version 3

Version 3 added the facility to process input files other than MT940 as input. This allows MT2OFX to be used to translate CSV and text files into OFX and QIF for use with Microsoft Money, Intuit Quicken, AceMoney and other financial programs.

 

Whereas MT940 is a more-or-less standardised format, CSV and text files offered by banks for download vary tremendously. The processing scripts for CSV and text files are invoked as the file is read, rather than as the output file is being produced.

 
V3.0.18, 12 May 2004.

  • Txn object now includes Category which can be set from scripting and is output in QIF files as the transaction category (L-line).

  • ParseLineDelimited now handles a trailing separator correctly.

 

Important Changes in Version 2.

Whereas Version 1 was only aimed at ABN AMRO Bank, Version 2 transformed MT2OFX into a more flexible product which can easily be adapted for additional banks and user preferences.

 

The multi-bank support includes rule-based detection of the originating bank of an MT940 file, and uses Microsoft's scripting technology to allow the automatic processing provided by the program to be changed and added to by users.
